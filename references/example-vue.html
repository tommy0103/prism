核心决策

迁移的核心问题有两个：使用什么 token 格式，以及客户端如何存储 token。

Access token 15 分钟过期；refresh token 单次使用，7 天滑动窗口。

倾向于使用 httpOnly cookie — 现有的 CSRF 中间件可以直接复用。

需要引入 Redis 或等价的 session 存储，增加基础设施复杂度。已排除

客户端 Auth 中间件 路由处理

JWT 校验是 CPU 密集但很快的操作。核心逻辑在 中。

export async function verifyAuth(req: Request) {
  const token = extractBearerToken(req);
  if (!token) throw new AuthError('missing_token');

  const payload = await verifyJWT(token);
  return { userId: payload.sub, scopes: payload.scopes };
}
